## 06장. 블로그 기획하고 API 만들기

### 6-1 사전 지식 : API와 REST API

**API**: 프로그램 간에 상호작용하기 위한 매개체  
**REST**: 웹 자원을 URI로 식별하고, HTTP를 통해 자원의 상태를 주고받는 API 방식  
**REST API**: REST 원칙을 따르도록 설계된 API

**REST API의 특징**  
서버/클라이언트 구조(로직 분리), 무상태(클라이언트 상태 저장X), 캐시 처리 기능, 계층화, 인터페이스 일관성  
-장점: 주소와 메서드만 보고 요청의 내용을 파악할 수 있음  
-단점: HTTP 메서드 개수에 제한이 있고, 표준 규약이 없음

**REST API 사용 규칙**  
-URL에는 동사를 쓰지 않고, 자원을 표시해야 함  
-동사는 HTTP 메서드로 표현 (GET-조회, POST-생성, PUT-전체 수정, PATCH-부분 수정, DELETE-삭제)
