### API

서로 다른 소프트웨어 애플리케이션이 데이터를 주고받는 기능을 공유할 수 있도록 연결해주는 소프트웨어 인터페이스

### REST API

자원의 이름을 통해 URI로 식별하고 HTTP로 해당 자원에 대한 CRUD 작업을 수행하는 규칙과 표준

서버/클라이언트 구조. 무상태, 캐시 처리 가능, 계층화, 인터페이스 일관성의 특징이 있다.

- 장점
    
    URL만 보고도 무슨 행동을 하는 API인지 명확하게 구분 가능
    
    HTTP 표준을 사용하는 모든 플랫폼에서 사용할 수 있다.
    
- 단점
    
    GET, POST 와 같은 방식의 개수에 제한이 있고, 설계를 위해 공식적으로 제공되는 표준 규약이 없음
    

### REST API 사용 방법

- URL에는 동사를 쓰지 말고, 자원을 표시해야 한다.
    
    동사를 사용할 경우 추후 개발 시 혼란을 줄 수 있기때문에 자원을 표시해  설계한다.
    
- 동사는 HTTP 메서드로 만든다.
    
    HTTP 메서드를 사용할때는 POST, GET, PUT, DELETE 등 동사로 표현한다.
