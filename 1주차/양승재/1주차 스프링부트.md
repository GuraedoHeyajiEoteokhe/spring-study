## 자바 기초

## 서버와 클라이언트

- 클라이언트
    
    사용자가 입력한 명령을 서버로 요청하는 프로그램
    
- 서버
    
    클라이언트에서 요청한 명령을 처리해 사용자에게 다시 보내주는 주체
    

## 데이터베이스

여러 사람에 의해 공유되어 사용될 목적으로통합하여 관리되는 데이터 집합

### 데이터 형태

- 정형 데이터
    
    미리 정해 놓은 형식과 구조에 따라 저장되도록 구성하여 고정된 필드에 저장된 데이터
    
    ex) 데이터베이스, 스프레드시트
    
- 비정형 데이터
    
    고정된 스키마 없이 자유로운 형식으로 이루어진 데이터
    
    ex) 이미지, 동영상, 오디오 파일, 텍스트 문서, 이메일 등
    
- 반정형 데이터
    
    정형 데이터와 같이 엄격한 구조는 없지만, 태그나 마크업 언어 등을 통해 일정 수준 구조를 갖춘 데이터
    
    ex) XML, JSON, HTML
    

### 데이터베이스의 종류

- RDB
    
    관계형 데이터베이스로 , 데이터를 행과 열로 이루어진 테이블로 관리(테이블 기반의 정형 데이터), 데이터 무결성, 신뢰성, 안정성을 보장함.
    
    가장 많이 사용하는 데이터 베이스
    
- NoSQL
    
    Not Only SQL로 비정형/ 반정형 데이터의 다양한 형식으로 데이터를 저장할 수 있고, RDB의 ACID보다는 확장성 및 가용성을 우선시 하는 경우가 많다.
    
- NewSQL
    
    RDB와 같은 테이블 기반 구조이지만 NoSQL과 RDB의 장점을 결합하여 뛰어난 수평 확장성을 가지고 있음, 하지만 상대적으로 높은 비용을 요구하기에 고성능 트랜잭션과 높은 확장성이 동시에 필요한 서비스에서 사용
    

### 현업에서 미리 만들어 둔 코드

- 라이브러리
    
    애플리케이션 개발에 필요한 기능인 클래스, 함수 등을 모아놓은 코드의 모음
    
    개발 중 import문을 사용하는 것이 라이브러리를 불러오는 것이다.
    
    프레임워크와 달리 개발자가 제어권을 가지고 작성 가능하다
    
- 프레임워크
    
    애플리케이션의 전체적인 구조와 흐름을 제공, 라이브러리와 달리 개발자가 프레임워크가 정해놓은 틀 안에서 자신의 코드를 추가
    

## 스프링 부트 기초

### 스프링

- 스프링
    
    자바 기반 웹 애플리케이션 개발을 위한 핵심 프레임워크, 의존성 주입과 AOP와 같은 핵심 기능을 제공
    
    개발자가 빈 객체 등록 및 의존성 설정을 직접 해야하며, XML 설정 등이 복잡하고 시간 소요가 큼
    
- 스프링부트
    
    스프링 프레임워크를 기반으로 개발 편의성을 높인 도구
    
    최소한의 설정으로 스프링 애플리케이션을 시작할 수 있게 해주는 ‘자동 설정’ 기능을 제공
    

스프링은 프로젝트를 띄우는 서버를 별도로 수동 설정 해야하지만 스프링부트는 내장형 서버를 제공해 별도의 설정 없이 사용가능(ex : 톰캣, 제티)

### 스프링 콘셉트

- 제어의 역전(IOC)
    
    프로그램의 제어권(객체의 생성, 호출, 생명주기 관리 등)을 프레임워크나 컨테이너와 같은 외부 환경에서 제어하여 객체가 스스로 자신에게 필요한 의존성을 생성하고 관리하는 대신 외부에서 이를 해결
    
- 의존성 주입(DI)
    
    제어권을 넘겨받은 외부 환경이 클래스에 필요한 의존 객체를 주입, 클래스 간의 결합도를 낮추고 유연성과 재사용성을 높이는데 사용
    
- 스프링 컨테이너
    
    빈을 생성하고 관리하고, 생성되고 소멸되기까지 생명주기를 관리
    
- 빈
    
    스프링 IOC컨테이너에 의해 생성되고 관리되는 자바 객체
    
    - Bean 구성요소
        - class
            
            Bean으로 등록할 Java 클래스
            
        - id
            
            Bean의 고유 식별자
            
        - scope
            
            Bean을 생성하기 위한 방법(Singleton, prototype)
            
        - constructor-arg
            
            Bean 생성 시 생성자에 전달할 파라미터
            
        - property
            
            Bean 생성 시 setter에 전달할 인수
            
    - Bean 어노테이션
        - @Component
            
            Component Scan을 사용해 @Component 어노테이션을 가지고있는 클래스를 찾아서 자동으로 Bean으로 등록해준다. 직접 개발한 클래스를 Bean으로 편리하게 등록하고자 하는 경우에 사용한다.
            
        - @Controller
            
            Component의 특수한 형태로, 웹 프레임워크에서 컨트롤러 역할을 하며 HTTP 요청을 받아서 처리하는 클래스임을 나타냄
            
        - @Service
            
            Component의 특수화 형태로, 비즈니스 로직을 구현하는 클래스
            
        - @Configuration
            
            Configuration이 붙어있는 클래스를 자동으로 빈으로 등록해주고 해당 클래스를 파싱해서 Bean이 있는 메소드를 찾아서 빈을 생성해주는 어노테이션
            
            - 개발자가 직접 제어가 불가능한 라이브러리를 활용할 때
            - 애플리케이션 전범위적으로 사용되는 클래스를 등록할 떄
            - 다형성을 활용하여 여러 구현체를 등록해주어야 할떄
            
            이떄 싱글톤으로 Bean을 생성하기 떄문에 Configuration을 사용한다.
            
- 관점지향프로그래밍
    
    핵심 관점과 부가 관점의 코드를 분리할 수 있게 해주어 프로그래머가 핵심 관점 코드에만 집중할 수 있게 해준다.
    
    코드의 가독성과 유지보수성 등을 높일 수 있습니다.
    
- 이식 가능한 서비스 추상화
    
    어떤 기술을 사용하든 일관된 방식으로 데이터베이스에 접근하도록 인터페이스를 지원하는 기술
    

### 스프링 부트 관련 설정

- @ComponentScan
    
    사용자가 등록한 빈을 읽고 등록하는 에너테이션
    
    @Component라는 애너테이션을 가진 클래스들을 찾아 빈으로 등록하는 역할
    
    Component 애너테이션만을 읽어오는게 아니라
    
    - Configuration : 설정 정보로 인식하고, @Bean 을 찾아 Singleton으로 관리
    - Repository : 접근 계층으로 인식하고, 데이터 계층의 예외를 spring 예외로 변환
    - Controller,RestController : 해당 클래스를 Controller로 인식
    - Service : 개발자 간의 Semantic annotation으로, 비즈니스 로직을 처리하는 역할
- @EnableAutoConfiguration
    
    스프링 부트의 자동 구성을 활성화하는 에너테이션으로 스프링 부트 서버를 실행할 때 스프링 부트 메타 파일을 읽고 정의된 설정들을 자동 구성
    

## 스프링 부트 구조

### 스프링 부트 계층

![image.png](attachment:d72837f8-f2aa-477c-9353-53fff9a93644:image.png)

- 프레젠테이션 계층
    
    HTTP 요청을 받고 이 요청을 비즈니스 계층(Service)로 전송하는 역할
    
    1. @Controller 애너테이션을 사용하여 빈에 등록
    2. @Autowired로  빈 Service를 주입해준다 
    3. GET 또는 POST 방식으로 Mapping 해준다
- 비즈니스 계층
    
    비즈니스 로직을 처리
    
    비즈니스 로직 : 서비스를 만들기 위한 로직
    
    1. @Service를 선언하여 Service로 비즈니스 계층 설정
    2. Autowired 로 빈 repository 를 주입
    3. repository에 있는 매서드를 호출
- 퍼시스턴스 계층
    
    모든 데이터베이스 관련 로직을 처리, 데이터베이스에 접근하는 DAO객체를 사용할 수 있다.
    
    1. 애너테이션을 이용해(NoArgsConstructor, AllArgsConstructor, Getter, Entity)을 선언해 Member class 선언
    2. Member안 Entity 애노테이션을 사용해서 테이블을 매핑
